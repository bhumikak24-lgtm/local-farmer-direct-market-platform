# Project Constitution: Local Farmer Direct Market Platform

## Core Principles
1. **Transparency**: All code and processes must be open and accessible.
2. **Quality**: Maintain high standards for code, documentation, and tests.
3. **Collaboration**: Foster an inclusive environment for all contributors.
4. **Compliance**: Strictly adhere to the Swecha GitLab Compliance standards.

## Development Rules
- Use AGPLv3 for all source code.
- Every feature must have a corresponding specification in `specs/`.
- All PRs must pass linting, formatting, and tests.
- Maintain 80%+ test coverage.
- Use semantic versioning and meaningful commit messages.

## Compliance Requirements
- ESLint, Prettier, Biome, Oxlint, and Knip must be configured and active.
- Secret scanning (Gitleaks) is mandatory for every commit.
- Automated changelog generation is required.
- GitLab CI must include all compliance stages.
